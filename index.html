Разработка программ

  • Постановка задачи это идея описанная на человеческом языке
  • Результатом разработки является код
  • Объектно ориентированный подход позволяет:
    • разобраться в постановке задачи
    • написать поддерживаемый код с возможностью повторного использования

Пример

  • Поиск космических аппаратов на марсе по фотографиям
  • Клиентская часть (frontend)
  • Название: AstroCrowd

Use cases (прецеденты)

Как использование системы обеспечивает ощутимый для пользователя результат?

Неправильно: авторизованным пользователям доступна функциональность системы

Правильно: пользователь может авторизоваться в системе для того чтобы получить доступ к ее функциям

AstroCrowd use cases

  1. Пользователь авторизуется в системе, после этого ему становятся доступны все ее функции
  2. Используя систему пользователь загружает фотографию с сервера, пытается определить видно ли на фотографии место посадки космического аппарата и сообщает свое решение системе
  3. Пользователь может просмотреть условия лицензионного соглашения
  4. Пользователь может просмотреть информацию о проекте и организаторах

Подробно главный use case

  1. Используя систему пользователь загружает фотографию с сервера
    • Если система определяет сбой во время загрузки фотографии она информирует об этом пользователя
  2. Пользователь принимает решение есть ли на фотографии следы космического аппарата
    • Если пользователь нашел что-то система предлагает ему отметить на фотографии заинтересовавшие его области
  3. Система сохраняет решение пользователя для этой фотографии на сервере
    • Если система определяет сбой во время сохранения решения она информирует об этом пользователя

Распределение обязанностей

  • Распределение обязанностей — это процесс определения какой класс выполняет какую работу
  • Существуют рекомендации как это делать (GRASP)
  • GRASP: General Responsibility Assignment Software Patterns — фундаментальные шаблоны распределения обязанностей

Шаблоны GRASP

Шаблон Information Expert

Какой наиболее общий принцип распределения обязанностей?

Назначить обязанность информационному эксперту — объекту, у которого имеется информация, достаточная для выполнения обязанности

Шаблон Information Expert

Необходимо сообщить пользователю общую площадь, которую он отметил для поиска. Кто должен ее расчитывать?

			Decision.prototype.getTotalS = function () { ... };
		
			Area.prototype.getS = function () { ... };

Шаблон Low Coupling

Как обеспечить простоту внесения изменений и повысить возможность повторного использования?

Распределить обязанности таким образом, чтобы степень связанности была низкой

Степень связанности объекта — это мера, определяющая с каким количеством других объектов он взаимодействует

Шаблон Low Coupling

Шаблон High cohesion

Как обеспечить контроль за сложностью?

Распределить обязанности таким образом, чтобы степень зацепления была высокой

Зацепление — это мера связанности и сфокусированности обязанностей класса

Шаблон High cohesion

Необходимо сообщить пользователю общую площадь, которую он отметил для поиска

			Decision.prototype.getTotalS = function () { ... };
		
			DecisionForm.prototype.showS = function () { ... };

Объектно-ориентированное проектирование

  • Строим модель предметной области
  • Объекты предметной области — основа программных компонентов
  • Распределяем обязаности, используя шаблоны проектирования
  • В процессе распределения обязанностей могут появиться новые программные компоненты

Как вности изменения в программу?

  • Внести изменения в прецеденты (use-cases)
  • Понять, как измиеняется предметная область
  • После этого станет ясно какие програмные компоненты затронут изменения
  • Распределить новые обязанности согласно GRASP
